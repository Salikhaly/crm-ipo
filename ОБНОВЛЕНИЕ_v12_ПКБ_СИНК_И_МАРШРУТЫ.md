# 🚀 Обновление v12 — автопередача из ПКБ, кредитная история, свои маршруты этапов

## ⚠️⚠️ ТРЕБУЕТСЯ МИГРАЦИЯ 015 + ENV-ПЕРЕМЕННАЯ ⚠️⚠️
1. **Supabase → SQL Editor → `migrations/015_pkb_sync_credits_routes.sql` → Run** (проверка вернёт 5 строк).
2. **Vercel → Settings → Environment Variables → добавить `PKB_API_KEY`** (любая длинная случайная строка) → Redeploy.
   Если не добавлять — используется существующий WEBHOOK_SECRET.

## 1. Автопередача лидов: анкета в ПКБ → лид в CRM сам 🔄

**Настройка (один раз):** ПКБ-приложение → Настройки → вкладка **CRM** → адрес
(`https://crm-ipo.vercel.app`) и API-ключ (тот же PKB_API_KEY). Пусто = выключено.

**Как работает:** менеджер создал анкету (или сделал «Импорт анкет») → клиент автоматически
появляется в CRM на этапе «Новый лид» с тегом `#пкб`: ФИО, ИИН, телефон, СРЗП → доход,
нагрузка/мес, нал. на руках → взнос, депозит, **полная кредитная история**, цель и ПКР — в ленту.
Если клиент уже есть (по телефону/ИИН) — обновляются цифры и добавляется запись в ленту, дубль не создаётся.
Ошибка сети НЕ мешает созданию анкеты (просто строка в логе). Endpoint: `POST /api/integration/pkb` (ключ в заголовке).

## 2. Кредитная история в карточке 🏦

- Импорт анкеты (кнопкой или автопередачей) теперь тянет **все кредиты** с листа «Кредиты»:
  активные (тип, банк, сумма, платёж, остаток, просрочка), завершённые (+цессионарий, дата), старые.
- Карточка → **Финансы → «Кредиты из ПКБ»**: таблицы активных и завершённых, итог платежей,
  кнопка **«В нагрузку/мес»** (обновляет поле одним кликом), ручное добавление/удаление строк.
- Экспорт «Анкета ПКБ (.xlsx)» пишет кредиты обратно построчно — круг замкнут (тест симметрии).

## 3. Свои маршруты этапов по программам 🗺 (главное!)

Панель техника → вкладка **«Маршруты»**: 7 групп (Полное, Онлайн, Доп. доход, Отбасы,
Госпрограмма, Поиск дома, Коммерческая) — у каждой **свой список этапов**, и у каждого этапа:
- **название**, **«что делать»** (инструкция менеджеру), **«что говорить»** (скрипт клиенту);
- добавить/удалить/поменять местами, всё редактируется и сохраняется (миграция 015);
- карточка клиента сразу использует ваши маршруты (степпер, чек-листы, гайды) — fallback на встроенные, пока не сохранили.

⚠️ Прогресс клиентов привязан к порядковому номеру этапа: тексты и названия правьте свободно,
новые этапы добавляйте **в конец** (об этом же предупреждает подсказка в редакторе).

## 4. Карточка клиента: всё видно сразу 👤

- **Шапка**: этап + тип договора + сумма + **ипотечная программа** (новое поле в Профиле, жёлтый чип 🏦).
- **Переуступка** (если включена) — чипы статусов прямо в шапке:
  `○/✓ Справка о задолж.` · `○/✓ Справка о переуступке` · `○/✓ Кредит закрыт`.
  Ставятся во вкладке Переуступка → блок «Статус переуступки» (3 больших чекбокса).

## 5. История по задачам 📜

Отметил задачу выполненной (или снова открыл) — в **Ленте** появляется запись
«✅ Выполнена задача: Позвонить — обсудить взнос» с автором и временем. Работает во всех
трёх местах: вкладка Задачи, задачи этапа сопровождения, правая колонка.

## Деплой
1. Миграция 015 + env PKB_API_KEY (см. выше) → git push (или мерж PR) → Vercel.
2. В ПКБ-приложении заполнить Настройки → CRM.
3. Проверить: создать анкету в ПКБ → лид появился в CRM с кредитами; админка → Маршруты →
   поменять текст этапа → сохранить → открыть карточку клиента этой группы → текст обновился;
   Переуступка → поставить галку → чип в шапке загорелся; выполнить задачу → запись в Ленте.
