# 🚀 Обновление v10 — импорт клиентов из Excel (анкета ПКБ)

Импорт сделан в **обоих** приложениях. Формат — анкета ПКБ (template.xlsx) + список база_анкет.xlsx.
Миграций НЕ требует.

## 1. Веб-CRM (crm-ipo)

Раздел **«Поиск / Все клиенты»** → кнопка **«Импорт»** (только руководитель/техник):
- Загружаешь один или несколько .xlsx.
- Поддержка двух форматов автоматически:
  - **Анкета ПКБ** (есть лист «Анкета») — 1 клиент на файл, можно выбрать пачку файлов;
  - **Список** (первый лист с заголовками ФИО/Телефон/ИИН/СРЗП/Нагрузка/Цель…, напр. база_анкет.xlsx).
- Предпросмотр таблицей (ФИО, телефон, ИИН, доход, нагрузка) → кнопка «Импортировать (N)».
- **Дубли** по телефону/ИИН пропускаются. Клиент создаётся: этап «Новый лид», тег `#импорт`,
  в ленту — заметка «📥 Импорт · Цель / ПКР / депозиты».

Что куда ложится: ФИО/ИИН/Телефон → профиль; СРЗП → доход; сумма платежей активных → нагрузка/мес;
наличные на руках → первоначальный взнос; депозит Н → депозит; цель/ПКР/БВ/ГП → заметка в ленте.

Технически: парсинг через SheetJS (`xlsx`), грузится **лениво** при открытии окна импорта —
на основной бандл не влияет. Логика маппинга вынесена в `lib/importAnketa.js` (22 теста).

## 2. Десктоп-приложение ПКБ (PKB_App_v8)

В блоке кнопок — **«📥 Импорт анкет»**:
- Выбираешь готовые анкеты .xlsx (можно много).
- Приложение читает данные клиента (те же ячейки, что и пишет) и добавляет строки в `база_анкет.xlsx`.
- Показывает сводку: добавлено / пустых / ошибок. Файл `база_анкет.xlsx` затем можно
  загрузить в веб-CRM кнопкой «Импорт» (формат «Список» распознаётся автоматически).

Изменён файл `app.py` (функция `read_anketa_xlsx` + метод `import_anketa` + кнопка).
Зависимостей не добавлял — используется уже установленный openpyxl.
