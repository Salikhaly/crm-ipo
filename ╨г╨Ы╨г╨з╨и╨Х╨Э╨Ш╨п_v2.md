# 🚀 Улучшения v2 — что добавлено

## ✅ Реализовано (8 из 9)

### №1 Сохранение расчётов в карточку клиента
Карточка клиента → Калькулятор → после расчёта кнопка **«Сохранить в карточку»**.
Расчёт с датой, автором и цифрами ложится в историю (видна под калькулятором).
Можно удалять. Хранится в БД (clients.saved_calcs).

### №2 PDF-отчёт клиенту
Кнопка **«PDF клиенту»** — открывает красивый печатный отчёт
(программа, взнос, займ, платёж, дисклеймер, контакты менеджера).
Браузер сохраняет как PDF (Ctrl+P → Сохранить как PDF срабатывает автоматически).
Работает без внешних библиотек.

### №3 Утренние WA-напоминания менеджерам
Каждый день в 9:00 (Астана) менеджерам с телефоном приходит в WhatsApp:
задачи на сегодня + просроченные, по их клиентам.
- Endpoint: /api/tasks/remind (защищён секретом)
- Расписание: vercel.json (Vercel Cron)
- Пауза 5 сек между менеджерами (anti-ban)
⚙️ Требуется: телефоны у менеджеров в админке + env CRON_SECRET (или используется WEBHOOK_SECRET)

### №4 Воронка конверсии + источники
KPI-страница теперь показывает:
- **Воронку по этапам** — сколько клиентов дошло до каждого, конверсия между этапами
  (красным подсвечены провалы <50% — там теряются клиенты)
- **Таблицу источников** — лиды/договоры/конверсия/выручка по Instagram, TikTok и т.д.
Считается на сервере по всей базе за выбранный период.

### №5 Автоответ новым лидам WA
Админка → вкладка WA-ответы → блок **«Автоответ новым лидам»**:
включатель + текст. Когда человек пишет впервые — CRM отвечает сама.
Существующим чатам не шлётся. Автоответ виден в истории чата как «Автоответ».
⚙️ Требуется: запустить миграцию 008, включить в админке.

### №7 URL-роутинг (лёгкий)
Раздел отражается в адресе (?p=calc). Кнопка «назад» браузера работает,
F5 не выкидывает на дашборд, ссылку на раздел можно скинуть коллеге.

### №8 Все клиенты в канбане
Раньше канбан видел первые 200 клиентов. Теперь автоматически догружаются
все страницы (до 1000) в фоне. Больше 1000 — подсказка использовать Поиск.

### №9 Мониторинг ошибок (без внешних сервисов)
Все ошибки фронтенда автоматически пишутся в Supabase → таблица error_logs:
текст, стек, страница, имя пользователя, браузер.
Больше не нужно просить скриншоты консоли — открой Table Editor → error_logs.
Защита от флуда: max 20 ошибок/мин, дедуп повторов.
(Хотел Sentry? Этот вариант работает сразу без регистраций. Sentry можно добавить позже.)

## ⏸️ №6 Realtime — отложен осознанно
Supabase Realtime требует anon-ключ на фронте, а у нас RLS блокирует
весь прямой доступ (безопасность через service_role на сервере).
Подключить Realtime = ослабить RLS или городить Broadcast-каналы.
Это отдельное архитектурное решение — не стал ломать безопасность ради скорости.
Polling работает, оптимизирован (пауза при скрытой вкладке).
