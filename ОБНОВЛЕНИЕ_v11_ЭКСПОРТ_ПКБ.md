# 🚀 Обновление v11 — экспорт в формате ПКБ (полный круг с template.xlsx)

Теперь и импорт, и экспорт работают с файлами приложения ПКБ. Миграций НЕ требует.

## 1. «Анкета ПКБ (.xlsx)» из карточки клиента

Карточка клиента → вкладка **Договор** → кнопка **«Анкета ПКБ (.xlsx)»**:
- скачивается **заполненный template.xlsx** — со всеми стилями/рамками оригинального шаблона
  (шаблон лежит в `public/pkb-template.xlsx`, заполняется через ExcelJS прямо в браузере);
- заполняются: ФИО (B5), ИИН (B6), телефон (F6), цель (H10 — из заметки импорта или тип договора),
  наличные на руках (H13 ← первый взнос), депозит (E18), СРЗП (B43 ← доход),
  нагрузка/мес — строкой «По данным CRM (итого)» в таблицу кредитов (E6).
- Файл читается ПКБ-приложением и обратным импортом CRM — проверено сквозным тестом.

## 2. Экспорт списка → база_анкет.xlsx (вместо CSV)

Раздел «Все клиенты» → кнопка **«Экспорт»** теперь выгружает **.xlsx в формате база_анкет**
(Телефон, Дата, Менеджер, ФИО, ИИН, ПКР, Нагрузка/мес, СРЗП, Цель, Нал. на руках, Депозит Н + Этап, Теги):
- открывается в Excel без танцев с кодировкой (это родной .xlsx);
- **этот же файл можно загрузить обратно кнопкой «Импорт»** — заголовки распознаются;
- работает и для выбранных строк в таблице (панель массовых действий → Экспорт).

## Технически
- `lib/exportAnketa.js` — чистый маппер (зеркало importAnketa), **9 тестов**, включая
  тесты симметрии: экспорт → импорт восстанавливает те же данные.
- Зависимость `exceljs@4.4.0` — грузится **лениво** при нажатии кнопки (бандл не растёт).
  ExcelJS выбран потому, что сохраняет стили шаблона (SheetJS CE их стирает).
- **Сквозной round-trip на реальных файлах**: ExcelJS заполнил настоящий template.xlsx →
  Python-читатель ПКБ-приложения (`read_anketa_xlsx`) прочитал → все 8 полей совпали.
