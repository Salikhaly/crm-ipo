# 🔍 Полный аудит и план редизайна — Ипотека CRM

Аудит проведён по реальному коду (pages/index.js, features/*, styles/globals.css), а не по общим словам.
Роли: UX-дизайнер · продакт · маркетолог · CRO · разработчик.

---

## Резюме (если читать одно)

Продукт **функционально сильнее**, чем выглядит. Внутри уже есть то, чего нет у amoCRM из коробки
(госпрограммы РК, маршруты сопровождения, скрипты менеджеру, WhatsApp с anti-ban). Три системные слабости:

1. **Продукт никак себя не продаёт** — сайта нет, есть только форма логина. Ноль конверсии в заявку.
2. **Выглядит «самодельно», а не «дорого»** — эмодзи вместо логотипа и иконок, 2 разных визуальных языка
   (синяя CRM и зелёный калькулятор), нет дизайн-токенов, разнобой радиусов и размеров шрифта до 8.5px.
3. **Карточка клиента перегружена** — 12 вкладок, важное (лента событий, задачи) спрятано, а не перед глазами.

Всё чинится итерациями без переписывания: система классов в globals.css уже приличная, архитектура позволяет.

## 1. UX — удобство использования

### Что уже хорошо (не трогать)
- Звонок/WA/статус звонка **в 1 клик из канбана** — лучше, чем в amoCRM.
- «Требует внимания сегодня» на дашборде — правильный фокус дня.
- «Следующий шаг» в карточке, гайды «что делать/что говорить», авто-задачи при смене этапа.
- Защита от дублей телефона, поиск в любом формате номера.
- Мобильная адаптация: bottom-nav, шторки, слайды WA — редкость для самописных CRM.

### Проблемы и решения

**UX-1. Создание клиента — длинная форма (~15 полей) [ВЫСОКИЙ]**
Новый лид = имя + телефон + источник. Остальное менеджер узнаёт позже.
→ Быстрое добавление: модалка из 3 полей («Имя, Телефон, Источник») + кнопка «Открыть карточку для заполнения».
Эффект: лид с телефона заносится за 10 секунд, меньше «занесу потом» (= потерянных лидов).

**UX-2. Поиск — отдельная страница вместо глобального [ВЫСОКИЙ]**
Сейчас: клик в раздел «Поиск» → ввод → карточка (3 шага + смена контекста).
→ Глобальный поиск в топбаре на каждом экране + горячая клавиша `Ctrl+K` (десктоп): имя/телефон/ИИН,
результаты выпадашкой, Enter — открыть. Раздел «Поиск/Все клиенты» превратить в «Список» (см. Структуру).
Эффект: самое частое действие менеджера — за 1 действие из любого места.

**UX-3. Карточка клиента: 12 вкладок [ВЫСОКИЙ]**
КИ, Отбасы, Переуступка, Анализ — отдельные вкладки, хотя заполняются один раз, а
Задачи/История, куда смотрят каждый день, спрятаны в конце списка.
→ Сгруппировать: **Профиль** (профиль+анализ) · **Финансы** (КИ+Отбасы+Переуступка аккордеонами) ·
**Договор и оплата** · **Сопровождение** · **Калькулятор** · **Файлы**.
Ленту событий и задачи вынести в **постоянную правую колонку** на десктопе (паттерн amoCRM) —
видно всегда, без кликов.
Эффект: -50% кликов в ежедневной работе с карточкой.

**UX-4. Смена этапа — из тёмного левого сайдбара [СРЕДНИЙ]**
Непривычное место, «меню навигации» по виду, а не статус.
→ Горизонтальный степпер-«ёлочка» над карточкой (как в amo): текущий этап подсвечен, клик — переход
(с уже готовыми проверками и причиной закрытия). Сайдбар карточки оставить только для «Назад» и ссылок.

**UX-5. Сохранение карточки — ручное [ВЫСОКИЙ]**
Кнопка «Сохранить» + диалог «выйти без сохранения» = менеджеры теряют правки.
→ Автосохранение с debounce 2–3 сек + индикатор «Сохранено ✓ / Сохраняю…» в шапке. Кнопку оставить как форс.
Эффект: ноль потерянных данных, минус целый класс жалоб.

**UX-6. Канбан: 10 колонок, горизонтальный скролл [СРЕДНИЙ]**
«Закрыто» и «Выдача» съедают место, активная работа — в скролле.
→ Свернуть «Закрыто» в узкую колонку-полоску (клик — развернуть); счётчик суммы этапа перенести в шапку колонки (уже есть) + фильтр «мои/все» для руководителя. Виртуализация карточек при >200 клиентах на этапе.

**UX-7. Нет отмены действий [СРЕДНИЙ]**
Перетащил не туда — ищи клиента заново.
→ Toast после переноса с кнопкой **«Отменить»** (5 сек): возврат этапа одним кликом.

**UX-8. Новичок vs опытный [СРЕДНИЙ]**
Подсказки-«хинты» видят все и всегда — опытных это шумит.
→ Кнопка «скрыть подсказки» (localStorage) + интерактивный тур первого входа (3–4 шага: канбан → карточка → WA → калькулятор). Тур уже в плане «не сделано» — поднять приоритет.

**UX-9. Задачи: нет календаря и переноса [НИЗКИЙ]**
→ Вид «Сегодня / Неделя / Просрочено» уже частично есть; добавить перенос задачи на другую дату в 1 клик (кнопки «+1 день», «+неделя»).

**UX-10. Распределение WA-лидов вручную [СРЕДНИЙ]**
Новые чаты назначает человек.
→ Round-robin автораспределение по активным менеджерам (настройка в админке, вкл/выкл).

## 2. UI — визуальный уровень

**UI-1. Дизайн-токены отсутствуют [ВЫСОКИЙ — фундамент всего]**
Цвета захардкожены в ~сотнях инлайн-стилей; радиусы 8/9/10/11/12/13/14/16/18/22px; размеры шрифта от 8.5px.
→ Ввести CSS-переменные в :root и планомерно переводить компоненты:
```css
:root{
  --brand:#2563eb; --brand-ink:#1e40af; --accent:#f59e0b;
  --ok:#10b981; --danger:#ef4444; --warn:#d97706;
  --ink:#0f172a; --ink-2:#475569; --ink-3:#94a3b8;
  --surface:#fff; --bg:#f1f5f9; --line:#e2e8f0;
  --r-sm:8px; --r-md:12px; --r-lg:16px;
  --fs-xs:11px; --fs-sm:12.5px; --fs-md:14px; --fs-lg:16px;
  --shadow-1:0 1px 4px rgba(15,23,42,.06); --shadow-2:0 6px 22px rgba(15,23,42,.1);
}
```
Правило: **минимальный текст 11px**, радиусы только из шкалы, тени только из шкалы.

**UI-2. Два визуальных языка [ВЫСОКИЙ]**
CRM — синяя (#3b82f6), калькулятор — зелёная (#1D9E75, свои C.*-стили). Ощущение «два разных продукта».
→ Калькулятор перевести на общие токены; зелёный оставить только семантически («одобрено», «выгода»).

**UI-3. Эмодзи вместо иконок и логотипа [ВЫСОКИЙ]**
🏠 как логотип, 🔴🟡📞 в заголовках, эмодзи в каждой вкладке — мило, но дёшево.
→ Tabler Icons уже подключены — использовать их везде (у них есть всё: ключ, дом, банк).
Эмодзи оставить только в WA-шаблонах сообщений (там уместно) и тостах-праздниках («🎉 одобрено»).

**UI-4. Перегруз декором [СРЕДНИЙ]**
Градиенты на шапках, тени на всём, цветные плашки в 4 слоя — глаз не отдыхает.
→ Правило «одна карточка — один акцент»: белые поверхности, 1 тонкая тень, цвет только у статуса/CTA.
«Дорого» = воздух и типографика, а не градиенты.

**UI-5. Скелетоны загрузки [НИЗКИЙ]**
«Загрузка KPI…» со спиннером → серые скелетоны карточек (16 строк CSS), ощущение скорости.

**UI-6. Тёмная тема [ПОЗЖЕ]**
Не критично для брокеров, но после перехода на токены будет стоить дёшево.

## 5. Главная страница сайта

**Главная находка аудита: сайта нет.** crm-ipo.vercel.app открывает форму логина.
Посторонний человек не узнаёт что это, для кого и сколько стоит. Конверсия в заявку = 0 по определению.

### Структура лендинга (по МАРКЕТИНГ.md, 1 страница)

1. **Hero (первые 5 секунд):**
   Заголовок: **«CRM для ипотечных брокеров Казахстана»**
   Подзаголовок: «Лид из WhatsApp → расчёт по госпрограммам → сопровождение до ключей. В одной системе.»
   CTA-1: «Посмотреть демо» (демо-стенд) · CTA-2: «Написать в WhatsApp».
   Справа — живой скриншот канбана (не стоковая иллюстрация).
2. **Боль → решение** — 3 карточки: «Лиды теряются в личных ватсапах» / «Расчёты в Excel» / «Менеджер не знает, что говорить» → как решаем.
3. **Демо в 30 секунд** — GIF/видео: сообщение в WA → лид появился → расчёт → PDF.
4. **Фичи под сегменты** — брокеру / риелтору / застройщику (из МАРКЕТИНГ.md).
5. **Госпрограммы** — логотипы-плашки: Наурыз, 7-20-25, Отбасы, 50/50… («мы считаем это из коробки»).
6. **Доверие** — цифры своего агентства («N клиентов в работе, M выдач через систему»), скрин KPI.
7. **Тарифы** — 3 карточки (Соло/Команда/Агентство) + «настроим за 1 день».
8. **FAQ + финальный CTA** (WhatsApp-кнопка, всегда видимая на мобиле).

Технически: статическая страница `/landing` в этом же Next.js (или отдельный проект), логин переезжает на `/app`.

### Экран логина (пока он — «лицо»)
Добавить: нормальный логотип вместо 🏠, одну строку «CRM для ипотечных брокеров РК»,
ссылку «Хочу такую CRM → WhatsApp». Стоит 30 минут, даёт канал заявок уже сейчас.

## 7. Конверсия (регистрация → покупка → удержание)

Сейчас воронки нет вообще: попасть в продукт можно только получив логин от техника.

1. **Заявка**: лендинг + WhatsApp-CTA (см. §5). Первая метрика — заявки в WA.
2. **Демо**: публичный демо-стенд `demo/demo` с фейковыми данными и баннером «Это демо. Хотите так же? → WA».
   Самый сильный конвертер для CRM: продукт показывает себя сам.
3. **Триал-виток**: «настроим за 1 день, 14 дней бесплатно» — снижает страх инсталляции.
4. **Активация** (чтобы купившие остались): онбординг-тур + чек-лист внедрения в админке
   («подключите WhatsApp → добавьте менеджеров → занесите 5 клиентов») с прогресс-баром.
5. **Удержание**: утренние WA-дайджесты руководителю (лиды/сделки за вчера — 1 сообщение),
   ежемесячный отчёт «сколько сделок прошло через CRM» = напоминание о ценности перед оплатой.

## 9. Недостающие функции

### Обязательно (без этого продукт не продать другим)
1. **Лендинг + демо-стенд** — продукт должен продавать себя (§5, §7).
2. **Глобальный поиск** (UX-2).
3. **Timeline в карточке** — единая лента: смены этапов, задачи, комментарии, WA-сообщения (данные уже есть, нужно собрать в одну ленту).
4. **Автосохранение карточки** (UX-5).
5. **Онбординг-тур + чек-лист внедрения** — иначе купившие не активируются.
6. **Экспорт клиентов в Excel** — руководители требуют всегда; без него «данные в заложниках».

### Желательно (отрыв от конкурентов)
7. Конструктор авто-задач/триггеров этапов в админке (свой Digital Pipeline).
8. Round-robin распределение WA-лидов.
9. Генерация договора из шаблона (docx с подстановкой ФИО/суммы) — сейчас договоры делают руками.
10. Слияние дублей (телефон уже ловим — добавить кнопку «объединить»).
11. Переключатель Канбан/Список + сохранённые фильтры.
12. Корзина вместо безвозвратного удаления клиента.

### Позже (когда пойдут продажи)
13. Клиентский портал: ссылка клиенту «статус вашей сделки» (этап, документы, следующий шаг) — вау-фактор и меньше звонков «ну что там?».
14. AI-помощник: черновик ответа в WA по контексту карточки; скоринг «горячести» лида.
15. PWA-push вместо звуковых уведомлений.
16. Мультитенантный SaaS + самостоятельная регистрация.
17. Интеграции: Kaspi-оплата счетов, календарь Google, телефония.

### Порядок внедрения (реалистичные волны)

- **Волна 1 «Лицо» (продаёт):** лендинг + демо-стенд + логотип/имя + экран логина. → появляются заявки.
- **Волна 2 «Скорость» (удерживает):** глобальный поиск, quick-add, автосейв, toast-отмена. → менеджеры счастливы.
- **Волна 3 «Карточка»:** группировка вкладок, timeline, степпер этапов. → уровень amoCRM+.
- **Волна 4 «Полировка»:** токены+единый стиль калькулятора, иконки вместо эмодзи, скелетоны, перфоманс.
- **Волна 5 «Продукт»:** онбординг, экспорт, конструктор триггеров, round-robin, портал клиента.

Каждая волна — отдельный zip с проверками (build + jest + undefined-скрипт), по нашим правилам.
