# 📌 РЕЗЮМЕ ПРОЕКТА — Mortgage CRM (прочти это в новом чате первым)

## Цель и идея
**Операционная система ипотечного брокера Казахстана** — не калькулятор для одного человека,
а рабочая система для всей команды: лид (WhatsApp / ПКБ-приложение / импорт) → расчёт по
госпрограммам РК → договор на услуги → сопровождение по своим маршрутам этапов → документы →
выдача → KPI. Долгосрочная цель — продавать другим брокерским агентствам РК
(лендинг /promo готов, тарифы Соло 15к / Команда 49к / Агентство 99к ₸/мес — гипотеза).
УТП против amoCRM: госпрограммы РК из коробки (Наурыз, Отбасы, 7-20-25…), расчёт ОПВ/СРЗП,
маршруты сопровождения со скриптами «что делать/что говорить», полный цикл с ПКБ-приложением.

## Продукты (два)
1. **Веб-CRM**: crm-ipo.vercel.app · GitHub Salikhaly/crm-ipo · автодеплой main → Vercel.
   Next.js 14.2 (pages router), React, Supabase (service_role только на сервере, RLS block-all),
   Green API (WhatsApp — ещё НЕ подключён), Google Drive (не настроен → fallback Supabase Storage).
2. **Десктоп ПКБ-приложение** (C:\Users\Шолпан\Downloads\PKB_App_v8, python/tkinter, отдельно от git):
   парсит PDF кредитного отчёта ПКБ и пенсионку → заполняет Excel-анкету (template.xlsx, листы
   «Анкета»+«Кредиты») → пишет базу_анкет.xlsx → **автопередаёт лида в CRM** (Настройки → CRM,
   POST /api/integration/pkb с x-api-key).

## Критичные правила работы (нарушение = краш прода)
1. После правок модулей: `npx next build` (env-заглушки), `npx jest` (сейчас **264** тестов),
   python undefined-скрипт (в ПАМЯТЬ_ПРОЕКТА-файле / истории чата).
2. reactStrictMode: false обязателен. Vercel: setTimeout после return не работает.
3. Новые колонки БД — через OPTIONAL_COLUMNS (lib/supabase.js) + add-функции: код обязан
   переживать неприменённые миграции.
4. В конце волны: zip без node_modules/.next/.git, инструкция ОБНОВЛЕНИЕ_vN.md, коммит в ветку,
   PR через GitHub REST API (gh CLI нет; токен: git credential fill).
5. Пользователь: русский, неформально, любит готовые zip и «сделай всё», отвечает на вопросы
   выбором. НЕ предлагать ИИ-фичи (сказал «без ИИ»).
